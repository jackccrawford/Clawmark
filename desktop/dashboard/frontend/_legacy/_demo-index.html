Geniuz Free — dashboard mocks

Four surfaces at four depths. Same content, same vocabulary, same product, scaled by screen real estate. Open source, designed-to-impress without becoming a workspace, intended to be the demo that creates the conversation. The companion commercial product (Geniuz RELAY) is a separate design pass focused on multi-station fabric and agent-fleet workflow tooling.

Design principles in the brief

  • Local · private · yours. The signature lives where first impressions form (dashboards, tooltip), not where status pulses (popup footer, which carries the more useful "Claude Desktop connected" instead). The privacy claim is fully earned with local LLMs (Ollama, llama.cpp, MLX); narrower but still defensible with Claude/cloud LLMs (queries route to the vendor as they would anyway, but Geniuz adds zero new third parties).
  • Lush, not minimal. Designed-to-impress at every depth. Geniuz Free is the demo workhorse — first impressions compound across weeks of campaign.
  • Consumer vocabulary. Memories, folder, conversations, find. Not stations, signals, threads, recall. Geniuz Free's translation layer over the open-source relay substrate.
  • Threading is a Free feature, made visible. Parent memories show a "↳ thread" affordance; follow-ups are indented with a connector elbow and a slightly muted gist treatment. The Free version carries real depth — date-descending, threads grouped, no nesting deeper than one level.
  • Memory-growth visibility is the stickiness flywheel. Sparkline on the hero stat, 14-day activity bars in the side panel. The user's accumulating corpus becomes the moat.
  • Fresh install is its own design moment. Surface 03 isn't an empty placeholder — it's a designed welcome with a 4-slide walkthrough and one self-aware seed memory. Every user meets this surface first; it has to introduce the product without explanation. The walkthrough is replayable from the sidebar at any time.
  • Header carries control, footer carries status. In the popup, Quit and Settings live in the header (with deliberate gap to prevent fat-finger Quit while reaching for the gear). Connection status and Dashboard CTA live in the footer. Memory list reclaims the height that would otherwise be eaten by stacked controls.
  • No fake Pro affordances. Free is complete and honest in its own right. RELAY upsell, if it exists, lives in About — never on the main surfaces.

Reference points

  • Stripe dashboard — depth + density, gradient confidence on hero stats
  • Linear / Notion — sidebar IA, layout density, typographic restraint
  • Cron / Things 3 — Mac-native polish translated through web tech
  • Raycast — screenshot-of-motion energy, microinteraction discipline

What's not here yet

  • Settings pane — gear icon click target. Canonical home for "Show walkthrough" replay, also where launch-at-login, recent-memories-count, and About would live. Mock as a fifth surface or as a route inside the dashboard window.
  • Data & export pane — the sovereignty surface. "Your memories are yours, here's how to take them anywhere." Worth its own screenshot.
  • Memory detail view — what happens when you click a row. Modal, side drawer, or inline expand. Where the full content (vs gist) lives along with thread context.
  • Zero-memory state — what if a fresh installer never runs the suggested first command? Different surface from "1-memory + carousel."
  • Inline help (ⓘ icons) — hover/click affordances next to titles like "Conversations 89" or "All searchable yes" to explain unfamiliar terms in-place. Reduces the need for a Help section.
  • Dark mode — explicitly out of scope. Light theme banks the brand identity for the launch campaign; dark mode arrives with RELAY's design pass.
  • Microinteractions / motion — static for now; gif-worthy hover/click states come after we lock visual language.
